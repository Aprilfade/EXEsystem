### 2. 智能课程推荐系统

**文件位置**：`exe-backend/src/main/java/com/ice/exebackend/service/RecommendationService.java`

#### 功能描述

基于用户考试成绩数据，智能分析薄弱科目，推荐相应课程进行学习。

#### 核心算法

**1. 薄弱科目分析**

```java
private Map<Long, SubjectPerformance> analyzeUserWeakSubjects(Long userId) {
    // 获取用户最近3个月的考试成绩
    List<BizExamResult> examResults = examResultMapper.selectList(
        new LambdaQueryWrapper<BizExamResult>()
            .eq(BizExamResult::getStudentId, userId)
            .ge(BizExamResult::getCreateTime, LocalDateTime.now().minusMonths(3))
    );

    // 按科目分组统计
    Map<Long, List<Double>> subjectScores = new HashMap<>();
    for (BizExamResult result : examResults) {
        BizPaper paper = paperMapper.selectById(result.getPaperId());
        if (paper != null) {
            double scoreRate = (result.getScore() * 100.0) / result.getTotalScore();
            subjectScores.computeIfAbsent(paper.getSubjectId(), k -> new ArrayList<>())
                         .add(scoreRate);
        }
    }

    // 计算平均分
    return calculateAveragePerformance(subjectScores);
}
```

**2. 推荐分数计算**

```java
private double calculateCourseScore(SubjectPerformance performance) {
    // 分数越低，推荐分数越高
    double baseScore = 100 - performance.averageScore;

    // 考试次数越多，置信度越高
    double confidenceFactor = Math.min(1.0, performance.examCount / 5.0);

    return baseScore * (0.7 + 0.3 * confidenceFactor);
}
```

**3. 推荐理由生成**

```java
private String generateCourseReason(SubjectPerformance performance) {
    String level;
    if (performance.averageScore < 60) {
        level = "需要重点提升";
    } else if (performance.averageScore < 75) {
        level = "有较大提升空间";
    } else {
        level = "可以进一步巩固";
    }
    return String.format("该科目平均得分率%.1f%%，%s",
                         performance.averageScore, level);
}
```

#### 推荐策略

1. **数据分析期**：最近3个月的考试成绩
2. **薄弱科目识别**：按平均得分率排序，取前3个薄弱科目
3. **课程匹配**：查询薄弱科目对应的课程
4. **智能排序**：按推荐分数排序（考虑薄弱程度和置信度）
5. **热门补充**：当推荐课程不足时，补充热门课程

#### 推荐效果

- 📊 **精准度**：基于真实考试数据，推荐准确度高
- 🎯 **个性化**：针对每个学生的薄弱环节
- 📈 **动态更新**：随考试数据自动更新推荐
- 🔄 **冷启动处理**：无考试数据时返回热门课程

```java
private List<CourseResult> searchCourses(String concept) {
    // 查询课程
    LambdaQueryWrapper<BizCourse> wrapper = new LambdaQueryWrapper<>();
    wrapper.like(BizCourse::getName, concept)
           .or()
           .like(BizCourse::getDescription, concept);
    wrapper.orderByDesc(BizCourse::getCreateTime);
    wrapper.last("LIMIT 10");

    List<BizCourse> courses = courseMapper.selectList(wrapper);

    // 计算相关度并排序
    for (BizCourse course : courses) {
        double relevance = calculateRelevance(concept, course);
        // ... 添加到结果列表
    }

    // 按相关度排序
    results.sort((a, b) -> Double.compare(b.getRelevance(), a.getRelevance()));

    return results;
}
```

**2. 相关度计算**

```java
private double calculateRelevance(String keyword, BizCourse course) {
    double relevance = 0.0;
    String lowerKeyword = keyword.toLowerCase();

    // 名称匹配（权重80%）
    if (course.getName() != null &&
        course.getName().toLowerCase().contains(lowerKeyword)) {
        relevance += 0.8;
    }

    // 描述匹配（权重20%）
    if (course.getDescription() != null &&
        course.getDescription().toLowerCase().contains(lowerKeyword)) {
        relevance += 0.2;
    }

    return Math.min(1.0, relevance);
}
```

#### 搜索特性

- 🔍 **模糊匹配**：支持名称和描述模糊搜索
- 📊 **相关度排序**：名称匹配优先级高于描述
- 🎯 **Top N结果**：限制返回10条最相关结果
- ⚡ **高效查询**：单次数据库查询完成

### 1. 用户认证机制优化

**文件位置**：`exe-backend/src/main/java/com/ice/exebackend/controller/BizPaperController.java`

#### 问题背景

之前AI试卷生成功能中，获取当前用户ID使用的是硬编码的临时值（返回1L），导致无法追踪实际用户操作。

#### 优化实现

```java
private Long getCurrentUserId() {
    try {
        // 从Spring Security上下文获取用户名
        String username = SecurityContextHolder.getContext()
                                               .getAuthentication()
                                               .getName();

        // 查询用户实体
        SysUser user = sysUserService.lambdaQuery()
                                     .eq(SysUser::getUsername, username)
                                     .one();

        if (user != null) {
            return user.getId();
        } else {
            logger.warn("当前用户不存在: {}", username);
            return null;
        }
    } catch (Exception e) {
        logger.error("获取当前用户ID失败", e);
        return null;
    }
}
```

#### 优化效果

- ✅ 获取真实登录用户ID
- ✅ 完善的错误处理和日志记录
- ✅ 支持审计和追踪
- ✅ 提升系统安全性

### 2. 登出流程统一优化

**文件位置**：`exe-frontend/src/stores/auth.ts`

#### 问题背景

教师端退出登录后直接跳转到登录页面（/login），而学生端跳转到Portal页面，用户体验不一致。

#### 优化实现

```typescript
async logout() {
    try {
        await logoutApi();
    } catch (error) {
        console.error("调用登出接口失败:", error);
    } finally {
        this.token = null;
        this.user = null;
        this.permissions = [];
        localStorage.removeItem('token');
        // 统一跳转到Portal页面
        window.location.href = '/portal';
    }
}
```

#### 优化效果

- ✅ 统一登出体验（教师端和学生端一致）
- ✅ 用户可重新选择登录角色
- ✅ 更好的用户引导流程
