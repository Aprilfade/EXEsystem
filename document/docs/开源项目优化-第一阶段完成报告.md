### 3. GitHub Actions 工作流

#### .github/workflows/ci.yml - 持续集成
```yaml
触发条件：Push 到 master/main/develop，PR 到 master/main
包含任务：
- 后端构建（Java 17 + Maven）
  - MySQL 8.0 服务
  - Redis 7.2 服务
  - 数据库初始化
  - 单元测试（待补充时可运行）
  - JaCoCo 测试覆盖率
  - 构建产物上传

- 前端构建（Node 20 + npm）
  - TypeScript 类型检查
  - ESLint 代码检查
  - 生产构建
  - 构建产物上传

- 代码质量检查
  - 提交信息格式检查（Conventional Commits）
  - 大文件检查（>10MB）
  - 敏感信息检查（密码泄露）

- 构建状态汇总
  - 汇总所有任务结果
  - 失败时退出并报错
```

```yaml
触发条件：Push 到 master/main，版本 tag，PR

  - Docker Compose 启动
  - 健康检查
  - 服务验证

- 构建状态通知
  - 成功/失败通知
```

  - 前端安全扫描

### 4. Issue 和 PR 模板

#### .github/ISSUE_TEMPLATE/bug_report.md - Bug 报告模板
```
包含字段：
- Bug 描述
- 复现步骤（1-2-3-4格式）
- 预期/实际行为
- 截图
- 环境信息（OS、浏览器、Java、Node.js、MySQL、Redis）
- 错误日志
- 检查清单（确保报告完整）
```

#### .github/ISSUE_TEMPLATE/feature_request.md - 功能请求模板
```
包含字段：
- 功能描述
- 问题背景（痛点分析）
- 解决方案
- 替代方案
- 使用场景（详细步骤）
- 界面设计（可选）
- 预期收益
- 技术考虑（可选）
- 检查清单
```

#### .github/PULL_REQUEST_TEMPLATE.md - PR 模板
```
包含字段：
- 变更说明
- 变更类型（bug/feat/docs等）
- 关联 Issue
- 变更详情（主要/次要改动）
- 截图对比（UI改动）
- 测试情况（环境、步骤、结果）
- 检查清单：
  - 代码质量（规范、注释、无调试代码）
  - 测试（单元测试、覆盖率）
  - 文档（README、CHANGELOG、Swagger）
  - Git（提交规范、无冲突）
  - 安全性（无敏感信息、安全漏洞）
```

   - ✅ 启用 Issues
